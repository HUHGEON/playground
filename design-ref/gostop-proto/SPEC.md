# 고스톱 연출 프로토타입 — 구현 스펙

출처: claude.ai/design 프로젝트 "고스톱 게임 UI 리팩토링"
파일: `고스톱 연출 프로토타입.dc.html`, `고스톱 애니메이션 분석.dc.html`

## 핵심 아키텍처 (가장 중요)
> **floor = 고정 6슬롯, 절대 안 움직임. 모든 이동은 flyer(별도 카드)로만.**
> index 재조정에 의한 바닥패 슬라이딩을 원천 차단.

- **고정 슬롯**: 바닥은 고정된 좌표의 슬롯들. 카드가 슬롯을 차지/비울 뿐, 슬롯 좌표는 불변.
  프로토타입 슬롯(620×430 좌표계, 카드 46×70):
  `[{165,88,-5},{409,88,5},{120,185,-7},{454,185,7},{165,282,-4},{409,282,6}]`
- **flyer**: 별도 absolute 카드. 스폰 → transform 패치(CSS transition .5s cubic-bezier(.2,.8,.3,1)) → drop.
  진짜 슬롯/더미 카드는 flyer가 **drop된 뒤에야** 나타남(= 미리 생성/번쩍 방지).
- **먹기(lift)**: 슬롯 카드를 flyer로 들어올리고 슬롯을 비움 → 더미로 이동 → drop → 더미 push.
- **뒤집기(reveal)**: flyer를 덱에서 스폰(face=back) → `ry:90`(rotateY) → `ry:0, face=front`(실제 플립) → 결과로 이동.

## 권장 페이싱 — 한 턴 ~2.6s 타임라인
- 0.0s 턴 시작: 낼 수 있는 손패 살짝 떠오름 + 족보 임박이면 골드 글로우
- 0.3s 손패→바닥: 낸 패가 바닥으로 미끄러져 매칭 패 위 정렬, 매칭 패에 링
- 0.8s 먹은 패→내 더미: 두 장 함께 족보 칸으로, 카운트+
- 1.3s 더미 뒤집기(플립): 보너스면 여기서 정지+콜아웃
- 1.8s 뒤집은 패 결과: 매칭→먹기 / 노매칭→바닥 깔림 / 특수→콜아웃+피뺏기
- 2.3s 점수 정산: 카운트업+토스트, 3점↑ 고/스톱 모달
- 2.6s 턴 넘김

## scEat (먹기·뒤집기) 시퀀스 — step(누적ms)
0 낸 패 flyer 스폰(hand) → 60 슬롯 위로 이동(.5s) → 480 매칭 슬롯 ring+flyer ring →
120 슬롯 카드 lift(flyer) → 40 둘 다 myTti로(scale .7) → 460 drop+pile push →
520 덱 flip 스폰(back) → 80 ry90 → 220 ry0 front(리빌) → 560 노매칭→빈슬롯 이동 → 440 drop+슬롯 set

## scBonus (보너스피) — 핵심: 정지+콜아웃
…먹기… → 520 덱 flip → 360 **보너스! scale1.5 위로+dim(B안)+콜아웃("보너스!","피 +2장")** →
900 **정적(인지 시간)** → 300 콜아웃 닫고 보너스 피 더미로 → 440 drop →
420 **보너스로 한 번 더 뒤집기(추가동작 구분)** → … 매칭 먹기 마무리

## scJjok (쪽·피뺏기)
낸 패 노매칭 깔림 → flip 같은 월 → 420 **"쪽!" 콜아웃**+ring → 760 깐패 되먹기+더미로 →
220 **상대 피 더미→내 더미로 날아오는 steal 애니** → drop

## scPpeok (뻑)
낸 패 바닥 겹침 → flip 같은 월 3장 → **"뻑!" 콜아웃**+3장 ring → 1100 **정지(왜 못 먹었는지)** → ring off

## scSsak (싹쓸이)
마지막 바닥 패 먹기 → 바닥 텅 → **"싹쓸이!" 콜아웃** + 상대 피 steal

## scJokbo (족보 임박+완성)
시작 시 내 띠 2장 + warn 배지("홍단 1장!"), 상대도 warn("청단 1장!") →
완성 패 내기 → 매칭 먹기 → **myWarn 제거** → **가운데 토스트("홍단","+3점") + countTo 점수(700ms)**

## scBomb (폭탄)
손 같은 월 3장 glow → 클릭 → 3장 함께 슬롯으로 + **"폭탄!" 콜아웃** → 4장 모두 내 더미로 →
패 모자라 **폭탄패 2장이 손에 glow** → 폭탄패 클릭(버튼 아님)으로 더미 한 장씩 뒤집기

## 변형 A vs B
- **A 절제형**: 작은 골드 토스트 칩 + 패 외곽 글로우. 인라인, 화면 안 가림. 빠른 게임용.
- **B 스포트라이트**: 핵심 순간 배경 dim + 패 중앙 확대 + 68px 서예체 콜아웃. 명확하나 개입적.

## 콜아웃/토스트
- callout(이벤트): 가운데. A=골드 칩(30px), B=대형 서예체(68px)+dim.
- toast(족보완성): 가운데 "족보 완성" + 카드들 + 족보명(34px) + 점수.
- warn 배지: 족보 칸에 "n장!" (ggWarn 펄스).

## 주요 키프레임
- flyer transition: `transform .5s cubic-bezier(.2,.8,.3,1)`
- ggGlow(손패 가능): translateY -12~-18 + 골드 ring 펄스 1.4s
- ggWarn: 골드 box-shadow 펄스 1.3s
- ggPop/ggToast: 콜아웃/토스트 등장 .35~.5s
- ring(강조): `0 0 0 2.5px #ffe680, 0 0 18px rgba(227,200,120,.7)`
